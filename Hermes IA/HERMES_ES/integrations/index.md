# Integraciones# IntegracionesHermes Agent se conecta a sistemas externos para inferencia de IA, servidores de herramientas, flujos de trabajo IDE, acceso programático y más. Estas integraciones amplían lo que Hermes puede hacer y dónde puede ejecutarse.:::consejo Comience aquí
Si solo tiene tiempo para configurar una integración, configure [Nous Portal](/integrations/nous-portal): un único inicio de sesión de OAuth cubre más de 300 modelos más las cuatro herramientas Tool Gateway (búsqueda web, generación de imágenes, TTS y automatización del navegador).
:::## Proveedores de IA y enrutamientoHermes admite múltiples proveedores de inferencia de IA listos para usar. Utilice `hermes model` para configurar de forma interactiva o configúrelos en `config.yaml`.- **[Proveedores de IA](/user-guide/features/provider-routing)**: OpenRouter, Anthropic, OpenAI, Google y cualquier punto final compatible con OpenAI. Hermes detecta automáticamente capacidades como visión, transmisión y uso de herramientas por proveedor.
- **[Enrutamiento de proveedor](/user-guide/features/provider-routing)**: control detallado sobre qué proveedores subyacentes manejan sus solicitudes de OpenRouter. Optimice el costo, la velocidad o la calidad con clasificación, listas blancas, listas negras y orden de prioridad explícito.
- **[Proveedores alternativos](/user-guide/features/fallback-providers)**: conmutación por error automática para proveedores LLM de respaldo cuando su modelo principal encuentra errores. Incluye respaldo del modelo primario y respaldo de tareas auxiliares independientes para visión, compresión y extracción web.## Servidores de herramientas (MCP)- **[Servidores MCP](/user-guide/features/mcp)**: conecte Hermes a servidores de herramientas externos a través del protocolo de contexto modelo. Acceda a herramientas de GitHub, bases de datos, sistemas de archivos, pilas de navegadores, API internas y más sin escribir herramientas nativas de Hermes. Admite transportes stdio y SSE, filtrado de herramientas por servidor y registro de recursos/indicaciones según la capacidad.## Motores de búsqueda webLas herramientas `web_search` y `web_extract` admiten ocho proveedores de backend, configurados mediante `config.yaml` o `hermes tools`:| Servidor | Var. ambiente | Buscar | Extracto | Arrastrarse |

| **Paralelo** | `PARALLEL_API_KEY` | ✔ | ✔ | — |
| **xAI** | `XAI_API_KEY` | ✔ | — | — |Ejemplo de configuración rápida:```yaml

```Si no se configura `web.backend`, el backend se detecta automáticamente a partir de cualquier clave API que esté disponible. Firecrawl autohospedado también se admite a través de `FIRECRAWL_API_URL`.## Automatización del navegadorHermes incluye automatización completa del navegador con múltiples opciones de backend para navegar por sitios web, completar formularios y extraer información:- **Browserbase**: navegadores en la nube administrados con herramientas anti-bot, resolución de CAPTCHA y servidores proxy residenciales.
- **Uso del navegador**: proveedor de navegador en la nube alternativo
- **CDP local de la familia Chromium**: conéctese a su navegador Chrome, Brave, Chromium o Edge en ejecución usando `/browser connect`
- **Local Chromium**: navegador local sin cabeza a través de la CLI `agent-browser`Consulte [Automatización del navegador](/user-guide/features/browser) para conocer la configuración y el uso.## Proveedores de voz y TTSTexto a voz y voz a texto en todas las plataformas de mensajería:| Proveedor | Calidad | Costo | Clave API |

Consulte [Voz y TTS](/user-guide/features/tts) y [Modo de voz](/user-guide/features/voice-mode) para obtener más detalles.## Integración de IDE y editor- **[Integración IDE (ACP)](/user-guide/features/acp)**: use Hermes Agent dentro de editores compatibles con ACP, como VS Code, Zed y JetBrains. Hermes se ejecuta como un servidor ACP, procesando mensajes de chat, actividad de herramientas, diferencias de archivos y comandos de terminal dentro de su editor.## Acceso programático- **[Servidor API](/user-guide/features/api-server)**: expone Hermes como un punto final HTTP compatible con OpenAI. Cualquier interfaz que hable el formato OpenAI (Open WebUI, LobeChat, LibreChat, NextChat, ChatBox) puede conectarse y utilizar Hermes como backend con su conjunto completo de herramientas.## Memoria y personalización- **[Memoria incorporada](/user-guide/features/memory)**: memoria persistente y seleccionada a través de archivos `MEMORY.md` y `USER.md`. El agente mantiene almacenes limitados de notas personales y datos de perfil de usuario que sobreviven a lo largo de las sesiones.
- **[Proveedores de memoria](/user-guide/features/memory-providers)**: conecte backends de memoria externa para una personalización más profunda. Se admiten ocho proveedores: Honcho (razonamiento dialéctico), OpenViking (recuperación por niveles), Mem0 (extracción en la nube), Hindsight (gráficos de conocimiento), Holographic (SQLite local), RetainDB (búsqueda híbrida), ByteRover (basado en CLI) y Supermemory.## Plataformas de mensajeríaHermes se ejecuta como un robot de puerta de enlace en más de 27 plataformas de mensajería, todas (`ha_list_entities`, `ha_get_state`, `ha_list_services`, `ha_call_service`). El conjunto de herramientas Home Assistant se activa automáticamente cuando se configura `HASS_TOKEN`.## Complementos- **[Sistema de complementos](/user-guide/features/plugins)**: amplíe Hermes con herramientas personalizadas, enlaces de ciclo de vida y comandos CLI sin modificar el código principal. Los complementos se descubren desde `~/.hermes/plugins/`, proyecto local `.hermes/plugins/` y puntos de entrada instalados por pip.
- **[Crear un complemento](/guides/build-a-hermes-plugin)**: guía paso a paso para crear complementos de Hermes con herramientas, ganchos y comandos CLI.## Capacitación y evaluación- **[Procesamiento por lotes](/user-guide/features/batch-processing)**: ejecute el agente a través de cientos de mensajes en paralelo, generando datos de trayectoria estructurados en formato ShareGPT para la generación o evaluación de datos de entrenamiento.---
